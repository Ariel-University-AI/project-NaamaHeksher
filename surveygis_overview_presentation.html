מודול 02 — למידת מכונה
מודל ML — ארכיטקטורה ופיצ'רים
Logistic Regression + Ridge Regression — מימוש NumPy טהור, ללא scikit-learn
🧮 שני מודלים מקבילים
Logistic Regression — מסווג
מסווג כל פרויקט: Profitable / Borderline / Not Profitable.
גבול סף: margin ≥ 20%.
Ridge Regression — מנבא כמות
מנבא את אחוז ה-margin הצפוי. רגולריזציית Ridge מונעת over-fitting על מדגם קטן.
למה NumPy בלבד?
Python 3.14 (סביבת הייצור) אינה תומכת ב-scikit-learn ו-scipy. המודלים ממומשים מאפס — gradient descent, normalization, cross-validation.
W = np.linalg.lstsq(X, y, rcond=None)[0] # Ridge: (XᵀX + λI)⁻¹Xᵀy W = np.linalg.solve(XtX + lam*I, Xty)

מדוע השיפור כל כך חד?
60 הפרויקטים שנוספו הכניסו טווח margin עד 576% שלא היה מיוצג. המודל "ראה" לראשונה פרויקטי RZ מהירים ולמד שהמרווח שם גבוה.
6 שגיאות שנותרו — ניתוח
כל 6 הטעויות הן פרויקטים עם margin 21–27% — קרוב לסף 20%. שגיאה שמרנית בלבד. אפס שגיאות קריטיות (לא ניבא רווח כשיש הפסד).
ניתן לכוונן עוד
שינוי MARGIN_THRESHOLD מ-20% ל-18% יסווג את 6 הפרויקטים כ-profitable. המחיר: מעט פחות שמרנות.
