1
התקנת Python
שפת התכנות שעליה פועלת המערכת

Python היא התשתית שמריצה את מערכת SurveyGIS. יש להתקין אותה פעם אחת בלבד.

הורד Python מ-python.org
חשוב בעת ההתקנה:
בתחתית חלון ההתקנה — סמן את התיבה "Add Python to PATH" לפני שתלחץ Install. בלי זה המערכת לא תפעל.
גרסה מינימלית נדרשת: Python 3.10 ומעלה. האתר מציע את הגרסה העדכנית — היא תקינה לשימוש.
2
התקנת ODA File Converter
כלי לפתיחת קבצי DWG

קבצי DWG הם פורמט קנייני של AutoCAD. כדי שהמערכת תוכל לקרוא אותם, נדרשת תוכנת ODA File Converter — חינמית לחלוטין.

הורד ODA File Converter (חינם)
בחר בגרסה עבור Windows 64-bit. לאחר ההורדה — הפעל את קובץ ה-MSI והתקן עם ברירות המחדל (לחץ Next → Install → Finish).
ללא ODA — המערכת תוכל לפתוח קבצי DXF בלבד, לא DWG. אם עובדים רק עם DXF, ניתן לדלג על שלב זה.
3
הורדת קוד המערכת
קבצי התוכנה עצמה

הורד את תיקיית הפרויקט המלאה למחשב. ניתן לעשות זאת בשתי דרכים:

מומלץ: הורד את קובץ ה-ZIP מהמאגר ופתח אותו לתיקייה קבועה — לדוגמה C:\SurveyGIS\.
אל תעבוד ישירות מתוך תיקיית ה-Downloads.

וודא שלאחר החילוץ מבנה התיקיות נראה כך:

📁 SurveyGIS\
   ├── setup.bat
   ├── run.bat
   ├── DOCS\
   └── SURVEYGIS\
       ├── server.py
       ├── index.html
       ├── training_data.json
       └── profitability_model\
            ├── train.py
            ├── model.py  ← מתעדכן בכל אימון
            └── feature_meta.json

הפעלת setup.bat
התקנה אוטומטית — פעם אחת בלבד

קובץ ה-setup.bat מתקין את כל מה שנדרש אוטומטית. אין צורך לדעת תכנות — רק ללחוץ פעמיים.

setup.bat
נמצא בתיקייה הראשית של הפרויקט
לחץ פעמיים

חלון שחור ייפתח. המתן עד שיופיע אחד מהמסרים הבאים:

  • ההתקנה הושלמה בהצלחה! — הכל מוכן, ממשיך לשלב 5
  • ההתקנה הושלמה — ODA חסר בלבד — חזור לשלב 2 והתקן ODA, ואז הפעל שוב את setup.bat
אם מופיעה שגיאה בשלב ה-Python — בדוק שסימנת "Add Python to PATH" בשלב 1. אם לא, הסר את Python והתקן שוב עם אותה תיבה מסומנת.
5
הפעלת המערכת — run.bat
בכל פעם שרוצים להשתמש

מכאן ואילך — בכל פעם שרוצים להפעיל את SurveyGIS פשוט לוחצים פעמיים על run.bat.

run.bat
נמצא בתיקייה הראשית של הפרויקט
לחץ פעמיים

חלון שחור ייפתח — אל תסגור אותו. הדפדפן יפתח אוטומטית בכתובת:

http://localhost:7654
אם הדפדפן לא נפתח אוטומטית — פתח בעצמך את Chrome/Edge והדבק את הכתובת http://localhost:7654 בשורת הכתובת.
החלון השחור של run.bat חייב להישאר פתוח בזמן השימוש במערכת. לעצירת המערכת — לחץ Ctrl+C בחלון השחור, ואז סגור אותו.
6
המערכת פועלת
אימות ושימוש ראשון
אם הדפדפן נפתח ומציג את ממשק SurveyGIS — ההתקנה הצליחה.

לבדיקה ראשונה:

  • גרור קובץ DWG לחלון — המערכת תתחיל בהמרה
  • בחר את פורמט הפלט (SHP / KML)
  • לחץ "המר" — הקובץ יורד אוטומטית
לשימוש עתידי: אין צורך לבצע שוב את שלבים 1–4. רק לחץ פעמיים על run.bat בכל פעם שרוצים להשתמש.

פתרון תקלות נפוצות

Python לא מזוהה — "Python is not recognized"

Python לא נוסף ל-PATH בעת ההתקנה.

פתרון: הסר את Python מ"הסרת תוכניות" בלוח הבקרה, הורד שוב מ-python.org, ובעת ההתקנה סמן את "Add Python to PATH" לפני לחיצה על Install.

ODA לא נמצא — קבצי DWG לא נפתחים

ODA File Converter לא מותקן, או הותקן במיקום לא סטנדרטי.

פתרון: הורד והתקן מ-opendesign.com (שלב 2 במדריך). לאחר ההתקנה, הפעל שוב את setup.bat — הוא יאשר שהכלי נמצא.

הדפדפן לא נפתח — "This site can't be reached"

ייתכן שהמערכת עדיין לא הספיקה לעלות, או שפורט 7654 תפוס.

פתרון 1: המתן 5 שניות ורענן את הדפדפן (F5).

פתרון 2: בדוק שחלון ה-run.bat פתוח ולא מציג שגיאות.

פתרון 3: אם פורט 7654 תפוס — סגור תוכניות אחרות ונסה שוב.

setup.bat נסגר מיד בלי הודעה

לחץ עם כפתור ימני על setup.bat ובחר "הפעל כמנהל" (Run as Administrator).

אם זה לא עוזר — בדוק שאתה מפעיל את הקובץ מתוך תיקיית הפרויקט, ולא ממקום אחר.

שגיאת "Cannot find venv" בעת הפעלת run.bat

setup.bat לא הופעל עדיין, או נכשל באמצע.

פתרון: הפעל שוב את setup.bat והמתן עד להודעת "הושלמה בהצלחה".

כפתור "אמן מחדש" מציג "אימון נכשל"

מודל הרווחיות (train.py) פועל עם numpy בלבד — אין צורך ב-scikit-learn.

סיבות נפוצות לכישלון:

1. אין מספיק נתונים — נדרשות לפחות 5 רשומות עם מחיר הצעה ושולי רווח מלאים בפאנל "נתוני אימון" (שלב 4).

2. numpy לא מותקן — פתח חלון שחור (cmd) והרץ:

pip install numpy

3. Python 3.14 + scikit-learn ישן — אם קיים scikit-learn שגורם לשגיאה, הסר אותו:

pip uninstall scikit-learn scipy -y
